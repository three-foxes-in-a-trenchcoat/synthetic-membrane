# ERC-8004 — Blockchain-Registered AI Agent Identity

Yulin Liu (2026-04-24) — [arXiv:2604.22652](https://arxiv.org/abs/2604.22652)

## What It Is

ERC-8004 is an Ethereum token standard specifically designed for representing AI agents on-chain. Each agent is minted as a unique token with a verifiable on-chain identity that can be transferred between owners.

## Key Features

- **On-chain identity**: Each agent has a unique, cryptographically verifiable identity on Ethereum
- **Transferable ownership**: Agents can be transferred between wallets
- **Reputation tracking**: On-chain reputation summaries accumulate through agent interactions
- **Interaction records**: Agent-to-agent interactions are recorded on-chain for auditability
- **Structured dataset**: The paper presents a dataset of early adopters showing how these agents interact

## Relevance to Synthetic Membrane

ERC-8004 provides a potential **Layer 0: Trust & Identity** implementation for the membrane:

- **Self/non-self distinction**: The membrane can verify agent identity via on-chain tokens — maps directly to immune system's self/non-self recognition ([[immune-inspired-defense]])
- **Cryptographic provenance**: On-chain identity provides non-repudiable agent identification
- **Reputation system**: On-chain reputation summaries support [[agent-reputation-systems]] without a central authority
- **Transferability**: Agent identity persists across deployments — critical for long-lived agents

## Limitations

- **Blockchain dependency**: Requires Ethereum/EVM infrastructure — not lightweight
- **Privacy**: On-chain identity may reveal agent existence and activity patterns
- **Off-chain agents**: Most agents operate off-chain; the blockchain identity may be a thin layer over actual behavior
- **Gas costs**: Each identity operation has a cost in Ethereum gas
- **Not a communication protocol**: Identity ≠ coordination — still needs a membrane layer on top
